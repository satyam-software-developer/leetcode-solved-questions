# [Tag Content Extractor](https://www.hackerrank.com/challenges/tag-content-extractor/problem?isFullScreen=true)
## Medium
<div class="challenge-body-html"><div class="challenge_problem_statement"><div class="msB challenge_problem_statement_body"><div class="hackdown-content"><svg style="display: none;"><defs id="MathJax_SVG_glyphs"></defs></svg><p>In a tag-based language like <em>XML</em> or <em>HTML</em>, contents are enclosed between a <em>start tag</em> and an <em>end tag</em> like <code>&lt;tag&gt;contents&lt;/tag&gt;</code>. Note that the corresponding <em>end tag</em> starts with a <code>/</code>.</p>

<p>Given a string of text in a tag-based language, parse this text and retrieve the contents enclosed within sequences of well-organized tags meeting the following criterion:</p>

<ol>
<li><p>The name of the <em>start</em> and <em>end</em> tags must be same. The HTML code <code>&lt;h1&gt;Hello World&lt;/h2&gt;</code> is <em>not valid</em>, because the text starts with an <code>h1</code> tag and ends with a non-matching <code>h2</code> tag.</p></li>
<li><p>Tags can be nested, but content between nested tags is considered <em>not valid</em>. For example, in <code>&lt;h1&gt;&lt;a&gt;contents&lt;/a&gt;invalid&lt;/h1&gt;</code>, <code>contents</code> is <em>valid</em> but <code>invalid</code> is <em>not valid</em>.</p></li>
<li><p>Tags can consist of any printable characters.</p></li>
